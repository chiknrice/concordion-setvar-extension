    • concordion-setvar-extension

    A concordion extension which allows creation of complex data structures in the spec and set it to variables before execute

    Project maintained by chiknrice Hosted on GitHub Pages — Theme by mattgraham

    Problem

    One limitation of concordion is setting a variable with a complex data structure such as a map without having to resort to calling execute commands to build the map in the fixture. Calling fixtures for setting up test data would mean state is required to be maintained within the fixture and this approach would certainly cause issues if we would want to execute tests in parallel. Although complex structures can be achieved with notations such as #{'key':'value'}, it would be helpful to show these test data as a table and instrument it with concordion commands.

    Another limitation related to the concordion:set command is that it can only set static values. It would be useful to have a concordion:echo command evaluate an expression and to be able to set the result to a variable using the concordion:set.

    Solution

    The set var extension solves these problems with a few awesome commands which build test data within the specification html (no need to call execute while setting up pre-conditions). This extension also introduces the concept of eval which evaluates expressions prior to building the complex data structures needed. As a bonus, a tooltip command is included to help make your specs more concise.

    Usage

    To use the extension, the fixture classes should be annotated with the following: 

        @RunWith(ConcordionRunner.class)
    @Extensions(SetVarCommandExtension.class)
    public class SampleFixture {
    }

    And the html spec files should declare the following namespace using any non conflicting prefix: 

        <html lang="en" xmlns:c="http://www.concordion.org/2007/concordion"
          xmlns:i="http://www.chiknrice.org/concordion">
