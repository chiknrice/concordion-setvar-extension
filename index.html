    concordion-setvar-extension

    One limitation of concordion is setting a variable with a complex data structure such as a map without having to resort to calling execute commands to build the map in the fixture. Calling fixtures for setting up test data would mean state is required to be maintained within the fixture and this approach would certainly cause issues if we would want to execute tests in parallel. Although complex structures can be achieved with notations such as #{'key':'value'}, it would be helpful to show these test data as a table and instrument it with concordion commands.

    Another limitation that this extension tries to solve is the concordion:set command being limited to static values. It would be useful to have a concordion:echo command evaluate expressions and to be able to set this to variables using the concordion:set.

    The set var extension solves these problems with commands which build test data within the specification html without the need for calling fixture methods. This extension also introduces the concept of eval which evaluates expressions prior to building the complex data structures needed.
